# AeroNet Control — макеты веб-интерфейса

Набор статических HTML-макетов для системы **AeroNet Control** — платформы управления управляемым роем беспилотных летательных аппаратов (БПЛА).

Проект выполнен в рамках практической работы по дисциплине **«Проектный практикум»**.

## О проекте

Макеты отражают ключевые входные и выходные потоки процесса планирования и сопровождения миссий роя: приём заявки, работа с картой и погодой, задание точек маршрута, формирование плана, мониторинг корректировок и отчётность.

Реализация — чистый HTML и CSS, без серверной части и сборки. Подходит для демонстрации структуры экранов, согласования с заказчиком и дальнейшей разработки.

## Состав макетов

### Входы

| Страница | Файл |
|----------|------|
| Заявка на миссию | `input-mission-request.html` |
| Картографические данные | `input-cartography.html` |
| Погодные данные | `input-weather.html` |
| Геоданные и аэронавигационные точки | `input-waypoints.html` |

### Выходы

| Страница | Файл |
|----------|------|
| План миссии с визуализацией маршрута | `output-mission-plan.html` |
| Отчёт о корректировке маршрута в реальном времени | `output-route-adjustments.html` |
| Отчёт о выполнении миссии | `output-mission-report.html` |
| Аналитическая сводка по эффективности маршрутов | `output-analytics.html` |

Обзор и навигация по всем разделам — в `index.html`.

### Шаблоны обмена данными

В каталоге `export/` приведены примеры экспорта в JSON и XML для двух потоков:

- `input-mission-request.json` / `.xml` — заявка на миссию;
- `output-route-adjustments.json` / `.xml` — отчёт о корректировках маршрута.

## Запуск

Откройте в браузере файл `index.html` (двойной щелчок или через контекстное меню «Открыть с помощью»).

Локальный веб-сервер не обязателен; при необходимости можно использовать, например:

```bash
python -m http.server 8080
```

и перейти по адресу `http://localhost:8080`.

## Структура репозитория

```
pp/
```
