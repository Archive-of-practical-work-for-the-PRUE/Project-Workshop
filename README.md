# AeroNet Control

Материалы практической работы по дисциплине **«Проектный практикум»** для системы **AeroNet Control** — платформы управления управляемым роем беспилотных летательных аппаратов (БПЛА).

Репозиторий объединяет два связанных направления:

1. **Макеты веб-интерфейса** — статические HTML-страницы экранов планирования и сопровождения миссий.
2. **Симулятор роевого интеллекта** (`drone-swarm-sim/`) — интерактивная веб-симуляция поведения роя дронов в реальном времени.

## О проекте

**AeroNet Control** предназначен для демонстрации и отработки сценариев работы с роем БПЛА: от приёма исходных данных и построения маршрута до мониторинга корректировок и аналитики.

Макеты интерфейса отражают ключевые входные и выходные потоки: заявка на миссию, карта, погода, точки маршрута, план с визуализацией, отчёты и сводки. Симулятор дополняет проект наглядной моделью поведения роя на основе классических правил роевого интеллекта.

## Макеты веб-интерфейса

Статические HTML и CSS без серверной части и сборки. Подходят для согласования структуры экранов с заказчиком и дальнейшей разработки.

### Входы

| Страница | Файл |
|----------|------|
| Заявка на миссию | `mockups/input-mission-request.html` |
| Картографические данные | `mockups/input-cartography.html` |
| Погодные данные | `mockups/input-weather.html` |
| Геоданные и аэронавигационные точки | `mockups/input-waypoints.html` |

### Выходы

| Страница | Файл |
|----------|------|
| План миссии с визуализацией маршрута | `mockups/output-mission-plan.html` |
| Отчёт о корректировке маршрута в реальном времени | `mockups/output-route-adjustments.html` |
| Отчёт о выполнении миссии | `mockups/output-mission-report.html` |
| Аналитическая сводка по эффективности маршрутов | `mockups/output-analytics.html` |

Обзор и навигация — в `index.html` (корень репозитория).

### Шаблоны обмена данными

В каталоге `export/` — примеры экспорта в JSON и XML:

- `input-mission-request.json` / `.xml` — заявка на миссию;
- `output-route-adjustments.json` / `.xml` — отчёт о корректировках маршрута.

### Запуск макетов

Откройте в браузере `index.html` в корне проекта.

При необходимости локального сервера:

```bash
python -m http.server 8080
```

Перейдите по адресу `http://localhost:8080`.

## Симулятор роевого интеллекта (`drone-swarm-sim/`)

Интерактивная веб-симуляция поведения роя дронов с алгоритмами **boids**:

- **Разделение (Separation)** — избегание столкновений между дронами;
- **Выравнивание (Alignment)** — согласование направления движения с соседями;
- **Сплочённость (Cohesion)** — удержание группы вместе.

Параметры меняются в реальном времени, результат сразу виден на экране.

### Возможности

- Количество дронов от 10 до 200;
- Поведение: максимальная скорость, радиус восприятия, веса разделения / выравнивания / сплочённости, притяжение к курсору;
- Визуализация: размер дронов, длина следа;
- Плавная анимация со счётчиком FPS;
- Адаптивный интерфейс.

### Управление

- Наведите курсор на область симуляции — дроны притягиваются к нему;
- Панель параметров — кнопка ⚙️ в правом верхнем углу;
- **Сбросить параметры** — возврат к значениям по умолчанию.

### Технологии

- HTML5 Canvas — отрисовка;
- JavaScript — логика симуляции;
- CSS3 — интерфейс.

### Алгоритм (каждый кадр)

У каждого дрона есть позиция, скорость, ускорение и история перемещения (след). На каждом шаге:

1. Определяются соседи в радиусе восприятия.
2. Считаются разделение, выравнивание и сплочённость.
3. При необходимости добавляется притяжение к курсору.
4. Скорость ограничивается максимумом, обновляется позиция.
5. При выходе за границы экрана дрон появляется с противоположной стороны.

### Запуск симулятора

Откройте `drone-swarm-sim/index.html` в современном браузере.

Сборка и запуск в Docker (nginx):

```bash
cd drone-swarm-sim
docker build -t aeronet-swarm-sim .
docker run -d -p 8080:80 --name aeronet-swarm-sim aeronet-swarm-sim
```

Симуляция будет доступна по адресу `http://localhost:8080`.
